### PlatformIO 快速入门

1. 首次使用需安装 **CH9102 USB 桥接** 驱动。
   * [Windows](https://www.wch-ic.com/downloads/CH343SER_ZIP.html)
   * [Mac OS](https://www.wch-ic.com/downloads/CH34XSER_MAC_ZIP.html)
2. 安装 [Visual Studio Code](https://code.visualstudio.com/) 和 [Python](https://www.python.org/)
3. 在 `Visual Studio Code` 扩展中搜索 `PlatformIO` 插件并安装。
4. 安装完成后，需要重启 `Visual Studio Code`
5. 重启 `Visual Studio Code` 后，选择左上角的 `文件` -> `打开文件夹` -> 选择 `LilyGo-Modem-Series` 目录
6. 等待第三方依赖库安装完成
7. 点击 `platformio.ini` 文件，在 `platformio` 栏中
8. Cancel the `;` symbol in front of the line `default_envs = T-SIM7600X` and use `T-SIM7600X` as the default environment
9. 取消注释 `src_dir = xxxx` 中的一行，确保只有一行有效。例如，要启用 `src_dir = examples/ATdebug`，删除前面的 `;` 符号并保存文件。现在将编译 ATDebug 示例。
10. 点击左下角的 (✔) 符号进行编译
11. Connect the board to the computer USB-C
12. 点击 (→) 上传固件
13. 点击（插头符号）监控串口输出
14. 如果无法写入，或 USB 设备持续闪烁，请查看下方的 **常见问题**

### Arduino IDE 快速入门

1. 首次使用需安装 **CH9102 USB 桥接** 驱动。
   * [Windows](https://www.wch-ic.com/downloads/CH343SER_ZIP.html)
   * [Mac OS](https://www.wch-ic.com/downloads/CH34XSER_MAC_ZIP.html)
2. 安装 [Arduino IDE](https://www.arduino.cc/en/software)
3. 安装 [Arduino ESP32](https://docs.espressif.com/projects/arduino-esp32/en/latest/)
4. 将 `lib` 目录中的所有文件夹复制到 `Sketchbook location` 目录。如何找到您自己的库位置，[请查看这里](https://support.arduino.cc/hc/en-us/articles/4415103213714-Find-sketches-libraries-board-cores-and-other-files-on-your-computer)
    * Windows: `C:\Users\%USERNAME%\Documents\Arduino\libraries`
    * MacOS: `$HOME/Documents/Arduino/libraries`
    * Linux: `$HOME/Arduino/libraries`
5. 打开相应的示例
    * 打开下载的 `LilyGo-Modem-Series`
    * 打开 `examples`
    * 选择示例文件并打开以 `ino` 结尾的文件
6. 在 Arduino IDE 中，在工具项目中选择相应的开发板，点击下方列表中的相应选项进行选择

    | 名称                                 | 值                                |
    | ------------------------------------ | ------------------------------------ |
    | Board                                | **ESP32 Dev Module**                 |
    | Port                                 | 您的端口                            |
    | CPU Frequency                        | 240MHZ(WiFi/BT)                      |
    | Core Debug Level                     | None                                 |
    | Erase All Flash Before Sketch Upload | Disable                              |
    | Events Run On                        | Core1                                |
    | Flash Frequency                      | 80MHZ                                |
    | Flash Mode                           | QIO                                  |
    | Flash Size                           | **4MB(32Mb)**                        |
    | JTAG Adapter                         | Disabled                             |
    | Arduino Runs On                      | Core1                                |
    | Partition Scheme                     | **Huge APP (3MB No OTA/1MB SPIFFS)** |
    | PSRAM                                | **Enable**                           |
    | Upload Speed                         | 921600                               |
    | Programmer                           | **Esptool**                          |

     * `Partition Scheme` 请根据实际应用选择。例如选择 **Huge APP (3MB No OTA/1MB SPIFFS)** , 更多分区方案请查看[这里](https://docs.espressif.com/projects/esp-idf/en/stable/esp32/api-guides/partition-tables.html)

7. In the open sketch, select the `utilities.h` file and uncomment the line `// #define LILYGO_T_SIM7600X`. Delete the `//` and save the file to use this board as the default configuration.

   ![board_select](../../../static/board_select.jpg)

8. 上传 sketch

> \[!IMPORTANT]
>
> 1. 请勿使用 USB-C 转 USB-C 为主板供电。请使用 USB-A 转 USB-C 为主板供电。Apple 电脑的 USB-C 端口无法直接为主板供电。
> 2. 无法上传任何代码？请查看下方的常见问题
> 3. 打开 Arduino IDE 时，会提示有新的库可以更新。请点击不更新。更新后无法保证能够编译、运行等。请确保开发板正常运行后再决定是否更新。要知道由于库的差异，更新后可能会出现问题，或者默认配置（如 TinyGSM）将被覆盖。
> 4. 如果更新后运行异常，请按照 Arduino 快速入门第 4 条删除 libraries 中的目录，并将 lib 中的文件夹重新拖入 libraries 目录。

#### ESP programming mode

<img src="../../../en/esp32/sim7600-esp32/images/TTL.png" alt="ESP programming mode" width="240" style="border: 0;"/>

* Switch P1~P2 to ON, and connect the modem's RX/TX to the ESP.
* Switch P4 to OFF to enable ESP control of the modem.

#### USB Modem mode

<img src="../../../en/esp32/sim7600-esp32/images/USB.png" alt="USB Modem mode" width="240" style="border: 0;"/>

* Switch P1~P2 to OFF, disconnect the modem's RX/TX to the ESP.
* Switch P4 to ON to enable automatic startup upon power-on, eliminating the need for ESP to control the modem's power-on.

* USB Modem mode is used to connect the modem to a PC to update the firmware or use dial-up Internet access.

| 名称            | GPIO NUM     | 可用 |
| --------------- | ------------ | ---- |
| Modem TX        | 27           | ❌    |
| Modem RX        | 26           | ❌    |
| Modem PWRKEY    | 4            | ❌    |
| Modem RING      | 33           | ❌    |
| Modem DTR       | 32           | ❌    |
| Modem Flight    | 25           | ❌    |
| Modem Status    | 34           | ❌    |
| Board LED       | 12           | ❌    |
| SD SCK          | 14           | ❌    |
| SD MISO         | 2            | ❌    |
| SD MOSI         | 15           | ❌    |
| SD CS           | 13           | ❌    |
| 电池 ADC 引脚 | 35           | ❌    |
| 太阳能 ADC 引脚   | 36           | ❌    |
| GPS Enabled     | Modem AUXVDD | ❌    |
| Default SDA     | 21           | ✅️    |
| Default SCL     | 22           | ✅️    |

* 连接网络时必须将 Modem Flight 引脚设置为高电平，否则将处于离线模式，无法连接蜂窝网络。
* 外设电源控制引脚。使用 SD 或 modem 时，必须将此 GPIO 设置为高电平，否则这些功能无法使用。
* ESP32 型号 GPIO33 及以上只能设置为输入，不能设置为输出

> \[!IMPORTANT]
> ESP32 版本由于 SD 卡使用 IO2 作为 CS，插入 SD 卡时无法在 Arduino IDE 中上传软件，因此上传新代码时必须取出 SD 卡
>
> GPIO4(PWRKEY),GPIO26(TX),GPIO27(RX), is exposed on the header pins. Do not connect any wires to the GPIO4 position on the header pins; keep it unconnected.
>
> GPIO25 (Modem FlyMode) is exposed on the header. Do not connect any wires to the GPIO25 position on the header; keep it disconnected.
>
> GPIO32 (Modem DTR) is exposed on the header. Do not connect any wires to the GPIO32 position on the header; keep it disconnected.
>
> GPIO33 (Modem RI) is exposed on the header. Do not connect any wires to the GPIO33 position on the header; keep it disconnected.

### ⚡ Electrical parameters

| 特性                        | 详情 |
| ------------------------------- | ------- |
| 🔗USB-C 输入电压            | 5V      |
| ⚡太阳能输入电压            | 5~6V    |
| ⚡USB-C/太阳能充电最大电流 | 500mA   |
| 🔋电池电压                | 3.7V    |
| 📍**VBUS** ( 排针)         | 5V      |
| 📍**VBAT** ( 排针)         | 4.2V    |

> \[!TIP]
>
> 1. 板载 JST2.0 接口仅用于连接太阳能板为电池充电，不能通过此接口为开发板供电
> 2. 建议使用输出电压为 6V 或 5V、功率约 5-10W 的太阳能板。太阳能只能为电池充电。没有电池时，太阳能板无法供电。
> 请注意，这只是一个建议。不一定要使用 5-10W 的太阳能板。1W 也可以使用。
> 太阳能板的大小决定了太阳能充电的电流和速度。请注意，根据环境不同，
> 每天的日照时间可能不超过一小时。此时，如果太阳能板太小且光线不足，则无法充电。
> 3. 将 VBUS 引脚连接到 USB-C 电源。如果 USB-C 存在，VBUS 电压等于 USB-C 电压。如果 USB-C 不存在，VBUS 上没有电压。
> 4. 将 VBAT 引脚连接到电池正极。此引脚可用于直接将外部电池连接到 VBAT 引脚。使用外部电池时，除非电池参数与外部电池相似，否则最好不要将电池连接到开发板的 18650 电池座。连接到 VBAT 引脚的电池可以使用板载充电管理进行充电。
> 5. VBUS 引脚是除电池外唯一为开发板供电的接口；没有其他方式。
> 6. USB/VBUS 引脚输入必须能够提供至少 2A 的峰值电流，且电压不得低于 5V，否则将触发低压自动关机。

### 常见问题

* **如果以上方法仍然无法上传 sketch，请按照以下方法测试 USB2TTL 通信是否正常。**
  1. 将 USB-C 连接到 PC
  2. 打开串口监视器，将波特率调整为 115200
  3. 按下开发板上的 RST 按钮重启
  4. 如果能看到启动信息，说明 USB2TTL 通信正常。如果串口监视器中没有任何显示，可能是 USB2TTL 桥接损坏

* **升级模块固件**
  1. [T-SIM7600X 升级文档](./upgrade/sim7600_upgrade.md)

* **无法检测到 SIM 卡？**
   1. 需要先将 SIM 卡插入开发板，然后再开机检测 SIM 卡。如果顺序颠倒，将报告无法检测到 SIM 卡。

* **无法上传任何 sketch，请手动进入上传模式。**
   1. 通过 USB 线连接开发板
   2. Use wires to connect GND and IO0 together
   3. Press RST button
   4. Release the RST button
   5. Disconnect IO0 from GND
   6. 上传 sketch

* **板载开关的作用**
  1. 使用内部电池组时，开关将正常工作以控制开发板的电源开/关。但是，如果使用外部电池组并连接到 VBAT 引脚，开关将被旁路，这意味着唯一的关机方式是断开电池。
  2. 开关仅用于电池供电，插入 USB 时无效

* **关于 VBUS 引脚**
  1. VBUS 引脚和 USB-C 在同一条线上。只有连接 USB-C 时，VBUS 才有电压
  2. 仅连接电池时，VBUS 没有电压输出
  3. 如果想连接外部电源而不连接 USB-C，VBUS 引脚是唯一的电压输入引脚。请注意，VBUS 引脚的最大输入为 5V，不要超过 5V

* **How to distinguish whether it has voice function?**
  1. If you need to make a voice call, you need to buy a SIM7600G with an audio decoder. You can check the red box in the figure below. If there is no chip here, there is no decoder version and the voice call function cannot be used.
  2. The figure below shows a version without a decoder. If there is a chip here, you can solder the microphone to the MIC+ and MIC- pads and the speaker to the SPK pad.

    <img src="../../../en/esp32/sim7600-esp32/images/decoder.jpg" alt="ESP programming mode" width="480" style="border: 0;"/>

  * 兼容驻极体麦克风和扬声器，但不兼容数字麦克风；仅兼容驻极体麦克风。

  <img src="../../../../docs/static/microphone.png" alt="microphone" width="240" style="border: 0;"/>

* 可以使用其他 GPS 天线吗？
  1. 外部 GPS 天线必须是 3.3V 供电的有源 GPS 天线。 For example, the [MK-76](https://www.rfmw.com/datasheets/sanav/mk-76.pdf) 支持 2.5-5.5V，兼容。大多数有源 GPS 天线支持 2.5-5.5V 供电。详情请参考 GPS 天线数据手册。
